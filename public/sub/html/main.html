Browsing in 3D

ThreeML aims to simplify the integration between the 2D and 3D web environment. This can be an exiting experience on flat screens, but also allows your business to prepare for the upcoming virtual reality glasses.

Why not have a 3D web that naturally extends the familiar experience we have on 2D sites?

A new perspective: your screen as camera

Many 3D sites aim to present a game-like reality in the browser. Although this is great for these applications and really show beautiful and realistic examples of what can be achieved today in a browser, it often doesn't follow the normal usage of the web. Over years of development of 'flat' pages the more or less standard behaviour of hyperlinks, mouse-cursors, tooltips etc. have made a convenient way for users to browse and interact with the environment. Furthermore it doesn't prepare a site for upcoming devices like immersive- or augmented reality glasses.

ThreeML is an open source project that aims to have the same kind of experience combined with the special possibilities that the 3D environment offers. It is based on the well known JavaScript module ThreeJS, and complements it by a declarative way to add much of the functionality, similar to how with Html most of the basic logic on a web page can be defined.

<three>
   <camera move="lookat"></camera>
   <group name="paintings1">
      <boxGeometry name="landscape" position="-1 0 0.2" rotation="0 0 0" scale="0.5 0.4 0.01">
         <meshphongmaterial id="picture1" color="1 1 1" url="./picture.jpg"></meshphongmaterial>
      </boxGeometry>

Those familiar with ThreeJS will recognize the tags used in the snippet above as being the names of objects when programming in JavaScript. Newbies can just copy some of these tags in an Html page, and find out what happens when changing the attributes. The ThreeML library also provides easy JavaScript methods to interact with the 3D world.

ThreeML is in a very early stage. It is a research project to investigate what techniques will improve 3D serving on the web, and implement them. The project code can be found and downloaded from GitHub.

Based on ThreeJS

ThreeJS.js is a well-known library for 3D presentation on the web. Their site displays many amazing examples that show what is possible today. The ThreeML module simplifies the initial setup of these scenes, and add many features that can be specified using a meta language with tags and attributes, just like is done using Html. Behind the curtain ThreeML supplies many common features that otherwise had to be accomplished by complex JavaScript/ThreeJS coding.

However, ThreeML exposes all objects in the familiar ThreeJS way, so if necessary they can be accessed by custom code and all features of ThreeJS can still be used.

Special methods for 2D interaction

To make it easy to interact between 2D (Html) pages, and the 3D environment, ThreeML has several specialized methods and objects. Most prominent are the windows that can be positioned anywhere in the 3D space, and which can show Html pages, either from inline Html, local web pages, or even external sites. Another common feature is to have these windows - or any other 3D object - switch between their original position to a presentation position in front of the camera, so the user can easily see them.

Support for .glb format

In recent years the open source .glb format has become an important standard for 3D models. Many programs, like Blender and Paint3D support the export to this file format. The file includes all necessary elements of the object, like the materials, textures and even animations. ThreeML can add these to a scene using one tag, and have the size, position and rotation specified by attributes.
