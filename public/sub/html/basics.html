The principles

Basic page

  • Html en Three tags can be mixed. The usual setup would be to have an initial page that mostly exists of a three segment. One or more floating frames (htmlPlaneGeometry) can be added to show Html content.
  • For the initial view the 3d space is defined by x, y, z coordinates, where x goes from negative left to positive right, y goes from negative down to postive up, and z goes from negative front (!) to positive back.
  • The camera - your screen - starts at position="0 0 0".
  • Position units can roughly be thought of as being in meters (m).
  • The initial page usually loads the threeml modules, as is demonstrated in the 'Hello Word' example. Although a floating frame (htmlPlaneGeometry) can initiate and use its own instance of threeml, it is better to reference the instance from the top window, especially if you want to access other objects using one of the specialiced threeml JavaScript methods.

3D objects, properties and inner tags

  • 3D objects like meshes (cubes, spheres, lights, models etc.) can be initiated by a tag that is often similar to their ThreeJS name. For example, the tag <boxgeometry> will put a box in the scene, having default properties. Attributes within the tag, like position , rotation and scale determine where and how the box should be placed.
  • The properties that represent a Vector3D should be specified with a single space between the 3 coordinate numbers. When only 1 number is specified, the Vector3D will be interpreted as having 3 similar coordinate numbers.
  • Enclosed within the object tag other special purpose tags can be specified. This includes the definition of the material of a mesh, or behaviors, like the possibility to have the object rotate itself at a certain speed and angle.

Group tags

  • Like with ThreeJS, <group> tags can be specied that can contain other objects. Objects within a group have their, position, rotation and scale set relative to these properties of the group object.
  • For several specific threeml functions, a group object is the 3D equivalent of an iframe in a (2D) Html page. This means for example that a new page with three tags can be opened, and the specfied objects will be loaded as childs of the group object.
  • Contrary to iframes in 2D, for group objects it is possible to choose whether to replace existing content (child objects) when loading a new page, or just to add extra content to the group object.
